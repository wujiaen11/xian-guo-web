# Vercel 部署指南

本指南将指导您如何在 Vercel 上部署商品管理系统，并添加自定义域名。

## 前提条件

- 已注册 Vercel 账号
- 已购买域名并完成备案
- 项目代码已准备就绪

## 步骤 1：部署网站到 Vercel

### 方法 A：通过 Vercel 控制台部署

1. **登录 Vercel 控制台**
   - 访问 [Vercel 官网](https://vercel.com/) 并登录您的账号

2. **创建新项目**
   - 点击控制台右上角的 "New Project" 按钮
   - 选择 "Import Project"
   - 选择 "Import Git Repository" 或 "Upload" 选项

3. **上传项目文件**
   - 如果选择上传方式：
     - 点击 "Upload" 按钮
     - 选择 `web` 目录中的所有文件（包括 index.html、css/、js/ 等）
     - 点击 "Deploy"

4. **配置部署设置**
   - 项目名称：可以设置为 `xian-guo-web`
   - Framework Preset：选择 "Other"
   - Build Command：留空（静态网站不需要构建）
   - Output Directory：留空（默认根目录）
   - 点击 "Deploy"

5. **等待部署完成**
   - Vercel 会自动部署您的网站
   - 部署完成后，您会看到一个成功页面，显示您的网站 URL

### 方法 B：通过 Vercel CLI 部署

1. **安装 Vercel CLI**
   ```bash
   npm install -g vercel
   ```

2. **部署项目**
   - 进入 `web` 目录
   ```bash
   cd web
   ```
   - 运行部署命令
   ```bash
   vercel
   ```
   - 按照提示完成部署过程

## 步骤 2：添加自定义域名

1. **进入项目设置**
   - 在 Vercel 控制台中，找到您刚部署的项目
   - 点击项目进入详情页
   - 点击左侧菜单中的 "Settings"
   - 选择 "Domains"

2. **添加自定义域名**
   - 在 "Add Domain" 输入框中输入您的域名（例如：`xianguo.site`）
   - 点击 "Add"

3. **配置 DNS 记录**
   - **获取 Vercel DNS 记录**：
     1. 在 Vercel 控制台的域名设置页面，找到您添加的域名
     2. 系统会显示需要添加的 DNS 记录，通常包括：
        - A 记录：将 @ 指向 Vercel 的 IP 地址（如 216.198.79.1）
        - A 记录：将 www 指向 Vercel 的 IP 地址（如 216.198.79.1）
        - 有时可能还需要添加 CNAME 记录
   - **登录域名注册商控制台**：
     1. 打开您购买域名的网站（如阿里云、腾讯云、GoDaddy 等）
     2. 登录您的账号
     3. 找到域名管理或 DNS 管理选项
   - **添加 DNS 记录**：
     1. 选择您要配置的域名
     2. 找到 DNS 解析设置
     3. 添加根域名记录（@）：
        - 记录类型：A
        - 主机记录：@（代表根域名）
        - 记录值：输入 Vercel 提供的 IP 地址
        - TTL：选择默认值（通常为 10 分钟或 1 小时）
     4. 添加 www 子域名记录：
        - 记录类型：CNAME（根据 Vercel 控制台的要求）
        - 主机记录：www
        - 记录值：输入 Vercel 提供的 CNAME 值（如 c9609653d6246d7e2.vercel-dns-017.com）
        - TTL：选择默认值
     5. 保存设置
   - **腾讯云 DNS 配置注意事项**：
     - 在腾讯云控制台中，添加 CNAME 记录时，确保记录类型选择为 "CNAME"
     - 记录值直接输入 Vercel 提供的完整域名，不需要添加额外的点
     - 腾讯云会自动验证记录格式，确保输入正确
   - **注意事项**：
     - 不同域名注册商的界面可能略有不同，但操作流程类似
     - 如果您之前有其他 DNS 记录，确保不要删除必要的记录
     - DNS 记录生效需要时间，通常为 1-24 小时

4. **验证域名**
   - 添加 DNS 记录后，点击 Vercel 中的 "Refresh" 按钮
   - 等待 Vercel 验证域名配置
   - 验证成功后，域名状态会显示为 "Connected"

5. **配置 HTTPS**
   - Vercel 会自动为您的域名配置 HTTPS
   - 等待证书颁发（通常需要几分钟）

## 步骤 3：验证部署

1. **访问网站**
   - 使用您的自定义域名访问网站（例如：`https://www.xianguo.site`）
   - 确保网站可以正常加载

2. **测试功能**
   - 测试商品管理、订单管理等功能
   - 确保所有功能正常运行

## 常见问题排查

### DNS 配置问题
- **问题**：域名状态显示 "Invalid Configuration"
- **解决方法**：
  1. 检查 DNS 记录是否正确添加
  2. 等待 DNS 记录生效（通常需要 1-24 小时）
  3. 再次点击 "Refresh" 按钮

### 网站无法访问
- **问题**：访问域名时显示错误
- **解决方法**：
  1. 检查 Vercel 部署状态
  2. 确保域名已正确配置
  3. 清除浏览器缓存后重试

### HTTPS 证书问题
- **问题**：网站显示不安全
- **解决方法**：
  1. 等待 Vercel 自动颁发证书
  2. 如果长时间未颁发，尝试移除域名后重新添加

## 注意事项

1. **Firebase 配置**：确保您的 Firebase 配置已正确设置，以便数据同步功能正常工作
2. **API 地址**：如果您使用了自定义 API，请确保 API 地址在 `index.html` 中正确配置
3. **域名备案**：确保您的域名已完成备案，否则可能无法正常访问
4. **定期备份**：建议定期导出数据，以防止数据丢失

## 后续维护

- **更新代码**：每次修改代码后，重新部署到 Vercel
- **监控网站**：使用 Vercel 的 Analytics 功能监控网站访问情况
- **安全更新**：定期检查并更新依赖项，确保网站安全
